[1단계] 공통 API 및 DB 구조 요청하기
웹과 앱이 함께 사용할 '두뇌'를 먼저 만듭니다.

AI 요청 프롬프트 예시:
"방문자 예약 시스템을 개발하려고 해. 웹과 모바일 앱에서 공통으로 사용할 API 서버가 필요해. **Next.js 14+ (App Router)**와 Prisma ORM, PostgreSQL을 사용할 거야.

먼저 방문자 예약 신청, 관리자 승인, 예약 조회 기능을 처리할 수 있는 DB 스키마(schema.prisma) 파일과, RESTful API (또는 Server Actions) 구조를 먼저 설계해 줘."

[2단계] 반응형(Responsive) 웹 UI 요청하기
데스크톱 웹과 모바일 화면 모두에 맞게 늘어나고 줄어드는 UI를 요청합니다.

AI 요청 프롬프트 예시:
"앞서 만든 API와 DB 스키마를 바탕으로, 방문자가 예약을 신청하는 페이지를 만들어줘.
조건: > 1. Tailwind CSS를 사용해서 **반응형(Responsive)**으로 짜줘.
2. PC화면에서는 넓게 보이고, 모바일 화면(스마트폰)에서는 한 줄로 깔끔하게 떨어지는 모바일 최적화 UI(Mobile-first)여야 해.
3. 입력 폼은 '방문자 이름, 연락처, 방문 목적, 방문 일시'를 포함해 줘."

[3단계] 안드로이드 앱(웹뷰) 패키징 요청하기
완성된 웹을 안드로이드 앱으로 변환하는 단계를 요청합니다.

AI 요청 프롬프트 예시:
"이제 위에서 만든 반응형 웹사이트 URL을 그대로 보여주는 안드로이드 앱을 만들려고 해.
가장 간단하게 구현할 수 있도록 Kotlin 기반의 Android WebView 프로젝트 구조를 짜줘.

앱이 켜지면 우리 웹사이트 주소(https://...)로 바로 접속되게 해줘.

뒤로가기 버튼을 누르면 앱이 꺼지지 않고 웹뷰 내에서 뒤로가기가 작동하게 코드 짜줘.

AndroidManifest.xml에 필요한 인터넷 권한 설정도 포함해 줘."
